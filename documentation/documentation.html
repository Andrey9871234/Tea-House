Introducere

Sa se implementeze o aplicatie Web care – via un API Web REST – simuleaza o ceainarie destinata in special studentilor informaticieni. Interactiunea cu "clientii" se va realiza din pagini Web publice gazduite pe un anumit server Web, pe baza campului 'Referer' din antetul unui mesaj HTTP. Aplicatia trebuie sa puna la dispozitia "clientilor" o serie de resurse identificate via URI precum '/intrare', '/meniu', '/preferinta/ceai/’ ‘/preferinta/ceai?de=tei’, '/vreau/suc', '/comanda/ceai/?fara=zahar', '/comanda/zacusca', '/nota', '/iesire' etc. (dati frau liber imaginatiei). Desi identificarea clientului se va realiza pe baza contului din URL-ul oferit de Referer, trebuie mentinut un nivel minimal contabil si informativ: ce produse sunt in stoc si la ce pret, cine si ce a comandat, care e valoarea totala a notei de plata, la ce ora s-a inregistrat o comanda, cine mai era atunci in ceainarie etc. Resursele sunt disponibile in cantitati finite, eventual ajustabile dinamic. "Filmul" ocuparii localului va putea fi expus, in mod dinamic, ca formate precum HTML, JSON si ca flux de stiri Atom. De asemenea, “proprietarul” ceainariei trebuie sa raspunda clientilor in mod "politicos” din punct de vedere HTTP -- exemplu: daca se recurge la 'GET /comanda/alcool', se va intoarce codul 403 Forbidden. A se lua in consideratie si alte exceptii.

Structura

Site-ul contine 7 pagini html. Navigatia se face usor de la o pagina la alta folosind o bara de navigatie care sta intotdeauna in partea de sus a paginii.

welcome.html

Pagina welcome este prima pagina pe care o intapina utilizatorul. Ea prezinta cateva informatii de baza precum: -locatia; -program; -cateva reviews; De asemenea prezinta o imagine de fundal si un mesaj de primire: "Welcome to Your Tea".

menu.html

Menu este pagina de unde utilizatorul se informeaza despre ce are ceainaria la dispozitie si ce anume contine din descrierea fiecarui obiect;

preferences.html

Pagina preferences prezinta o filtrare specializata bazata pe istoricul de cumparaturi ale userului prezentandui acestuia o lista cu obiectele achizitionate de acesta in alte sesiuni;

order.html

Pagina order este zona de completare a comenzii, actionand ca un cos de cumparaturi si proces de finalizare a comenzii printr-un formular.

Occupancy.html

Pagina Occupancy da detalii despre locurile libere ramase si numarul de clienti actuali in ceainarie.

Admin.html

Pagina pentru adminul site-ului de unde vor putea fi inserate noi alimente spre vanzare.

exit.html

Pagina exit este pagina informationala dupa finalizarea plasarii unei comenzi specificand diferite detalii precum numarul comenzii, data plasarii si timpul estimat al ridicarii comenzii.

Folderul assets

Aici sunt toate resursele care tin de imagini.

Folderul stylesheets

Aici sunt toate resursele care tin de aspectul paginilor html. Fiecare pagina are propriul fisier .css care asigura ca site-ul sa fie responsiv pe mai multe rezolutii. Aici avem navbar.css si item.css facute special pentru a nu creea prea mult cod redundant, izoland stilizarea repetitiva intr-un fisier separat ce doar trebuie importat.
